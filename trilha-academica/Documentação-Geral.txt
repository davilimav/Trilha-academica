## 1. **VISÃO GERAL DO SISTEMA**

O **Trilha Acadêmica** é uma plataforma web de **recomendação personalizada de cursos e trilhas de aprendizado**. Seu objetivo é ajudar estudantes e profissionais a descobrir conteúdos educacionais alinhados com seus interesses, nível de conhecimento e objetivos de carreira.

O sistema combina:
- **Backend em Node.js** com banco de dados MySQL para gerenciar usuários, progresso e avaliações.
- **API de recomendação em Python** (Flask + scikit-learn) que utiliza similaridade de cosseno para sugerir trilhas.
- **Frontend responsivo** com HTML, CSS (arquitetura de variáveis globais) e JavaScript puro.

---

## 2. **PRINCIPAIS FUNCIONALIDADES**

### **2.1. Autenticação e Perfil**
- Cadastro de novos usuários (nome, e‑mail, senha).
- Login com token JWT (expira em 7 dias).
- Recuperação de senha por e‑mail (Nodemailer).
- Upload de foto de perfil (multer, armazenamento local).
- Edição de informações pessoais: telefone, idade, cidade, estado, faculdade, semestre, bio.

### **2.2. Questionário de Interesses**
- O usuário escolhe áreas de conhecimento e cursos específicos de seu interesse.
- As respostas são salvas na tabela `questionario_respostas`.
- O sistema nunca força o usuário a responder novamente – apenas se ele optar por refazer o questionário (botão na área de configurações).

### **2.3. Recomendação de Trilhas**
- Os dados do questionário são enviados para a API Python.
- O algoritmo calcula o **nível de conhecimento médio** (1‑3) e a **similaridade** entre o perfil do usuário e 15 trilhas predefinidas.
- As recomendações são exibidas no dashboard, com percentual de compatibilidade.

### **2.4. Catálogo de Cursos
- Áreas, subáreas e cursos organizados hierarquicamente.
- Visualização de detalhes do curso: descrição, tópicos, projetos, pré‑requisitos, carga horária, certificado.
- Botão **“Marcar como concluído”** – registra o curso na tabela `cursos_concluidos`, concede pontos, atualiza progresso e desbloqueia conquistas.

### **2.5. Progresso e Gamificação**
- Tabela `progresso_usuario` armazena: nível atual, pontos totais, horas estudadas.
- Conquistas automáticas ao concluir 1, 3, 5 cursos.
- Currículo completo do aluno (visualização + download PDF) com dados pessoais, formação, cursos concluídos, conquistas e estatísticas.

### **2.6. Painel Administrativo**
- Acesso restrito a usuários com `is_admin = 1`.
- Estatísticas gerais: total de usuários, questionários respondidos, cursos concluídos, usuários ativos.
- Gráficos (Chart.js) de top especializações e nível médio por área.
- Listagem de usuários com seus níveis de conhecimento.

### **2.7. Currículo em PDF**
- O aluno pode baixar um PDF com todas as suas informações (html2pdf.js).

## 4. **PREPARAÇÃO DO AMBIENTE (PASSO A PASSO)**

> **Pré‑requisitos:**  
> - Node.js (versão 18 ou superior) instalado.  
> - Python 3.10 ou superior instalado.  
> - MySQL Server instalado e rodando.  
> - Editor de código (VS Code recomendado).

### **4.1. Clonar / Obter o projeto**
```bash
cd C:\Users\SeuUsuario\Desktop
git clone https://github.com/seu-repositorio/trilha-academica.git
cd trilha-academica
```

### **4.2. Configurar o banco de dados**
- Acesse o MySQL (Workbench ou terminal):
```sql
CREATE DATABASE trilha_academica;
USE trilha_academica;
```
- Execute o script de criação das tabelas (fornecido separadamente ou contido no arquivo `database.sql`).
- As principais tabelas são: `users`, `questionario_respostas`, `cursos_concluidos`, `progresso_usuario`, `conquistas`.

### **4.3. Configurar variáveis de ambiente**
Crie um arquivo `.env` na raiz do projeto com o seguinte conteúdo (ajuste conforme seu ambiente):
```env
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=sua_senha
DB_NAME=trilha_academica
JWT_SECRET=uma_chave_muito_segura
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=seu_email@gmail.com
EMAIL_PASSWORD=senha_app
PORT=3000
```

### **4.4. Instalar dependências do Node**
```bash
npm install
```

### **4.5. Instalar dependências do Python**
Abra um segundo terminal:
```bash
cd python-recomendacao
pip install flask flask-cors numpy scikit-learn
```

### **4.6. Popular o banco (opcional, mas recomendado)**
Execute o script de população para criar usuários fictícios, questionários, cursos concluídos, etc.:
```bash
node popular-banco-completo.js
```

### **4.7. Criar usuário administrador**
```bash
node criar-admin-correto.js
```
(O script criará o admin com e‑mail `admin@trilhaacademica.com` e senha `admin123`.)

## 5. **EXECUÇÃO DO SISTEMA**

### **5.1. Iniciar os servidores**
Você precisará de **dois terminais abertos** simultaneamente.

**Terminal 1 – Backend Node:**

**Terminal 2 – API de Recomendação (Python):**

```
*Saída esperada:*  
```

### **5.2. Acessar a aplicação**
- Abra o navegador e digite: `http://localhost:3000`
- Use as credenciais de admin ou crie um novo usuário.

### **5.3. Fluxo típico de uso**
1. **Login** → se nunca respondeu o questionário, é redirecionado para o **questionário**.
2. **Resposta do questionário** → escolha áreas e cursos de interesse.
3. **Dashboard** → visualiza recomendações de trilhas, estatísticas de progresso.
4. **Cursos** → navega pelo catálogo, acessa detalhes e conclui cursos.
5. **Currículo** → edita informações pessoais e baixa PDF.
6. **Admin** (se for administrador) → acessa gráficos e gerencia usuários.

## 6. **GUIA DE PERSONALIZAÇÃO VISUAL (PARA O COLEGA DESIGNER)**

Todo o estilo visual é controlado pelo arquivo **`frontend/css/trilha-global.css`**.  
As variáveis CSS permitem alterar cores, tamanhos, espaçamentos e sombras globalmente.

### **6.1. Alterar a cor principal do site**
No arquivo `trilha-global.css`, localize:
```css
--cor-primaria: #16a34a;
--cor-primaria-escura: #15803d;
--cor-primaria-clara: #22c55e;
```
Basta mudar o valor hexadecimal para a cor desejada (ex.: azul `#3b82f6`).

### **6.2. Mudar o arredondamento dos cantos**
```css
--borda-raio-sm: 8px;   /* botões, inputs */
--borda-raio-lg: 16px;  /* cards, containers */
```

### **6.3. Ajustar espaçamentos**
```css
--espaco-md: 16px;
--espaco-lg: 24px;
```

### **6.4. Alterar fontes**
```css
--fonte-principal: 'Segoe UI', system-ui, sans-serif;
```

### **6.5. Personalização por página (se necessário)**
Cada página HTML possui uma seção `<style>` específica para ajustes finos. Para mudar, por exemplo, o tamanho do título na página inicial (`index.html`):
```css
.hero-text h1 {
    font-size: 48px;
}
```

### **6.6. Como testar as alterações**
1. Salve o arquivo CSS.
2. Atualize a página no navegador (F5).
3. As mudanças refletem instantaneamente.

## 8. **SOLUÇÃO DE PROBLEMAS COMUNS**

| Problema                                      | Provável causa e solução                                                             |
|-----------------------------------------------|---------------------------------------------------------------------------------------|
| **Erro `ER_CON_COUNT_ERROR` / conexão MySQL** | MySQL não está rodando. Inicie o serviço ou verifique credenciais no `.env`.          |
| **API Python não retorna recomendações**      | Servidor Python não está rodando. Execute `python app.py` na pasta `python-recomendacao`. |
| **Botão “Marcar concluído” não aparece**      | Verifique se o arquivo `curso-detalhe.html` está atualizado (versão com botão).       |
| **Perfil não salva informações**              | Confirme se o `getProfile` está retornando os campos extras (telefone, idade etc.). Veja item 9.1. |
| **Admin Panel não carrega gráficos**          | As tabelas `user_especializacoes` ou `user_stats` podem não existir. Atualize o `adminController.js` para usar `cursos_concluidos` como fallback (conforme versão entregue). |
| **Login redireciona para questionário sempre**| O campo `questionario_respondido` não está sendo marcado. Rode: `UPDATE users SET questionario_respondido = 1 WHERE email = '...';` |
| **Erro `Cannot access 'API_URL' before initialization`** | A ordem das variáveis no script da página está incorreta. Use o código corrigido do `admin-panel.html` fornecido. |

## 9. **MANUTENÇÃO E MELHORIAS FUTURAS**

### **9.1. Adicionar novos campos ao perfil**
- Altere a tabela `users` (adicione as colunas).
- Atualize a query `SELECT` em `getProfile` (authController.js).
- Adicione os campos ao formulário de edição em `perfil-completo.html`.

### **9.2. Incluir novas trilhas de recomendação**
- Edite o dicionário `trilhas_config` no arquivo `python-recomendacao/app.py`.
- Recomenda‑se também adicionar os cursos correspondentes no catálogo (`data/cursosConteudo.js` e `cursoController.js`).

### **9.3. Melhorar a segurança**
- Implementar refresh token.
- Adicionar rate‑limiting nas rotas.
- Usar HTTPS em produção.

### **9.4. Funcionalidades sugeridas**
- Avaliação com estrelas para cursos.
- Comentários dos usuários.
- Geração de certificado em PDF ao concluir uma trilha completa.
- Dashboard com mais gráficos de progresso pessoal.

O **Trilha Acadêmica** é um sistema maduro, com arquitetura cliente‑servidor, pronto para ser utilizado em ambientes educacionais.
